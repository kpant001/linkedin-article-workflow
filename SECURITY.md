# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, please email: **pantkapil34@gmail.com**

Include:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

## Response Timeline

- **Acknowledgment**: Within 48 hours
- **Assessment**: Within 1 week
- **Fix**: Depending on severity, within 1-4 weeks

## Scope

This project is an agent skill definition (prompt engineering). Security concerns primarily relate to:

- Prompt injection vulnerabilities in agent instructions
- Information leakage through agent behavior
- Unintended actions triggered by crafted inputs

## Supported Versions

| Version | Supported |
|---------|----------|
| 1.x.x | Yes |
| < 1.0 | No |
